# 커뮤니티 신고 처리 운영 가이드 (SOP)

본 문서는 커뮤니티 내 사용자 생성 콘텐츠(UGC)에 대한 신고/차단/숨김 운영 기준을 정의합니다.

## 1) 적용 범위

- 대상: 커뮤니티 게시글, 댓글
- 신고 채널:
  - 앱 내 신고 기능 (게시글/댓글)
  - 이메일 접수: `vividxxxxx@gmail.com`
- 관련 기능:
  - 신고(Report)
  - 숨김(Hide post)
  - 작성자 차단(Block user)

## 2) 운영 역할

- **모더레이터(운영자)**
  - 신고 접수 확인
  - 위반 여부 판정
  - 조치 실행 및 상태 업데이트
- **개발 담당자**
  - 정책/룰 반영
  - 로그/증적 보존 지원
  - 반복 위반 패턴 기술 대응

## 3) SLA (권장)

- 접수 확인(Initial triage): **24시간 이내**
- 일반 위반 건 처리: **72시간 이내**
- 중대 위반(혐오, 위협, 성적 착취, 자해/타해 조장): **24시간 이내 우선 처리**

## 4) 신고 처리 절차

1. 접수
   - 앱 내 신고 또는 이메일 신고 확인
2. 분류
   - 스팸/광고
   - 혐오/차별
   - 음란/선정성
   - 괴롭힘/위협
   - 기타
3. 판정
   - 정책 위반 여부 판단
4. 조치
   - 게시글/댓글 삭제 또는 유지
   - 필요 시 작성자 제한(차단/추가 제재 검토)
5. 기록
   - 처리 결과/근거/시각 기록
6. 재검토
   - 반복 신고 사용자/콘텐츠 패턴 월간 리뷰

## 5) 조치 기준 (간단 버전)

- **즉시 삭제 대상**
  - 노골적 성적 콘텐츠
  - 폭력/테러/자해 조장
  - 특정 집단에 대한 명백한 혐오/위협
- **제한/경고 대상**
  - 반복 스팸/광고
  - 괴롭힘성 표현 반복
- **유지 가능**
  - 정책 위반 근거가 불충분한 경우 (기록 후 모니터링)

## 6) 사용자 보호 기본 원칙

- 신고자는 앱 내에서 숨김/차단 기능으로 즉시 자기 보호 가능
- 차단 시 해당 작성자의 게시글/댓글은 노출되지 않음
- 신고 상세 내용은 운영 목적 외 사용 금지

## 7) 로그/증적 관리

- 신고 ID, 대상 ID, 신고 사유, 처리자, 처리 시각, 최종 조치 기록
- 분쟁 가능성이 있는 건은 최소 90일 증적 보관 권장

## 8) App Review 제출 시 명시 항목

- UGC 보호 기능 제공 여부:
  - 게시글/댓글 신고 가능
  - 게시글 숨김 가능
  - 작성자 차단 가능
  - 제한어 필터 적용
- 운영 문의 채널:
  - `vividxxxxx@gmail.com`

## 9) 정기 점검 체크리스트

- 주 1회: 신고 처리 대기 건 확인
- 월 1회: 반복 위반 패턴과 필터 룰 업데이트
- 릴리즈 전: 신고/숨김/차단 동작 회귀 테스트
